# Human-matched remote runs — validity

**Gates (human-matched):** image tracking ≥0.70 and parse quality ≥0.97 for **valid**; **borderline** if tracking ≥0.50 but not fully valid; **invalid** if tracking &lt;0.50. The **word sensitivity** metric from [`validity_gates.R`](analysis_pipe/src/validity_gates.R) is **not used** here: each stimulus is paired with a **single** pseudo-word, so the benchmark-style word-sensitivity rate is structurally zero. Raw `word_sensitivity_rate` is still written to the CSV for transparency.

**Inputs:** `eval_mode=human_matched` CSVs under `results/model.results/human_matched/` (typically v1 + v2 from the same batch). For the **combined** table, `stim_id` is disambiguated as `{stim_pkg}/{stim_id}` so numeric IDs from v1 and v2 are not merged.

**Regenerate:**

```bash
python scripts/compute_validity_human_matched.py
python scripts/update_models_validity_md.py \
  --skip-word-sensitivity-gate \
  --csv results/data/model_validity_summary_human_matched.csv \
  --md interpret/human_matched_validity.md
```

Per-package summaries (v1 only / v2 only) are written to `results/data/model_validity_summary_human_matched_v1.csv` and `_v2.csv` by the compute script.
